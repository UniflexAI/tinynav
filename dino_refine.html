5. Training and Deployment Path

  1. Label retrieval samples in WebUI and export reviewed pairs.
  2. Fine-tune LoRA using reviewed pairs.
  3. Merge LoRA into base DINOv2 model.
  4. Export merged model to ONNX.
  5. Build TensorRT engine with make dinov2.

Commands:

  • python3 tool/train_dinov2_lora.py --config <config.yaml>
  • python3 scripts/merge_dinov2_lora.py --base_model facebook/dinov2-base --lora_path <lora_ckpt> --out_dir <merged_model>
  • python3 scripts/export_dinov2_lora_onnx.py --model_path <merged_model> --out_onnx /tinynav/tinynav/models/dinov2_base_224x224_fp16.onnx
  • cd /tinynav/tinynav/models && make dinov2

6. Risks and Controls

  • Small reviewed dataset can overfit. Control: expand positives and hard negatives.
  • TRT conversion can fail on graph mismatch. Control: validate ONNX and tensor names early.
  • Runtime compatibility depends on last_hidden_state output shape and naming.
